## Database Design for 인공지능 트렌드 분석 및 핵심 요약 일일 발송 툴

### 목표 요약
- 외부 소스(기사, 논문, 블로그, 트윗 등)를 수집/저장
- 자동/수동으로 트렌드를 추출(클러스터링, 토픽), 각 트렌드에 대한 핵심 요약 생성
- 사용자별 구독(주제 필터, 빈도: 매일/매주 등) 및 채널별 발송(이메일/슬랙 등)
- 발송 기록 저장, 미리보기 API, 수동 재발송/재생성 지원
- 검색(텍스트, 임베딩 유사도) 및 성능 확장 고려

### Tables
- users
  - id UUID PRIMARY KEY
  - email VARCHAR NOT NULL UNIQUE
  - name VARCHAR
  - created_at TIMESTAMP DEFAULT now()
- sources
  - id UUID PRIMARY KEY
  - name VARCHAR NOT NULL
  - type VARCHAR(32) -- blog, paper, twitter, rss, api
  - url TEXT
  - metadata JSONB
  - created_at TIMESTAMP DEFAULT now()
- articles (수집된 원문/요약/메타)
  - id UUID PRIMARY KEY
  - source_id UUID REFERENCES sources(id) ON DELETE SET NULL
  - external_id TEXT -- 원출처 고유 id (중복방지)
  - title TEXT
  - content TEXT
  - excerpt TEXT
  - url TEXT
  - published_at TIMESTAMP
  - language VARCHAR(8)
  - authors JSONB
  - metadata JSONB
  - tsv tsvector -- full-text index column (computed)
  - embedding vector(1536) NULL -- pgvector 사용(크기 예시)
  - created_at TIMESTAMP DEFAULT now()
- ai_trends (추출된 트렌드 / 토픽)
  - id UUID PRIMARY KEY
  - title VARCHAR -- 대표 문구
  - description TEXT -- 자동/수동 요약
  - topic_keywords TEXT[] -- 키워드
  - prominence FLOAT -- 점수(예: 언급량, 중요도)
  - first_seen TIMESTAMP
  - last_seen TIMESTAMP
  - metadata JSONB
  - created_at TIMESTAMP DEFAULT now()
  - updated_at TIMESTAMP DEFAULT now()
- article_trend_links (기사 ↔ 트렌드 연결, 가중치)
  - id UUID PRIMARY KEY
  - article_id UUID REFERENCES articles(id) ON DELETE CASCADE
  - trend_id UUID REFERENCES ai_trends(id) ON DELETE CASCADE
  - relevance FLOAT -- 0..1
  - matched_terms TEXT[]
  - created_at TIMESTAMP DEFAULT now()
  - UNIQUE(article_id, trend_id)
- trend_summaries (트렌드별 일일 요약 버전, 재생성 가능)
  - id UUID PRIMARY KEY
  - trend_id UUID REFERENCES ai_trends(id) ON DELETE CASC
